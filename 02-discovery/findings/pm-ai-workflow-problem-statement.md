# Problem Statement: AI-Accelerated PM Workflow & File Organisation

**Date:** 2026-04-24
**Author:** Chunchun Tong
**Context:** Plaid project — developed during folder organisation and inbox-triage skill design session

---

## Problem Framing Narrative

**I am:** A Product Manager who has meaningfully accelerated their individual output by adopting AI tools — researching faster, producing documents and slides quicker, and iterating on ideas in real time.
- I move fast across many workstreams simultaneously — strategy, discovery, delivery, comms — often in a single day
- I adopted AI tools specifically to reduce friction, not add it
- I care about outcomes, not process — file organisation feels like overhead, not work
- I work primarily alone on this project today, but expect to onboard teammates in the future

**Trying to:** Stay in a flow state while producing high-quality work, and be able to find and reuse what I've made without losing time to archaeology.

**But:**
- AI tools accelerate output so much that files accumulate faster than I can organise them
- There's no moment in my workflow where I'm naturally prompted to file something correctly — saving a file is an afterthought
- When I need to find something later, I can't remember which tool generated it, what it was named, or where it landed
- The folder structure I intend to follow exists in my head, but not reliably in my hands at save time

**Because:** AI-assisted work creates a volume and velocity of artefacts that outpaces the organisational habits designed for slower, more deliberate document creation. The faster I work, the worse the mess gets.

**Which makes me feel:** Productive in the moment, but increasingly anxious and frustrated when I can't find things — like I'm building on sand. There's also a creeping guilt that the mess will become someone else's problem when the team grows.

---

## Context & Constraints

- Works across multiple file types generated by different tools (Cowork, browser downloads, VS Code exports, PowerPoint, markdown editors) — no single save location
- File organisation happens after the fact, not at the moment of creation
- The intended folder structure is new and not yet habitual — no muscle memory built yet
- No enforcement mechanism exists at the OS or app level to guide file placement
- Future state: team will grow, making current disorganisation a compounding risk

---

## Final Problem Statement

**A PM using AI tools to accelerate their work needs a way to keep their project artefacts organised without interrupting their flow, because AI-assisted output accumulates faster than any manual filing habit can keep up with — leaving them unable to find, reuse, or share their own work, and anxious about the growing mess.**
